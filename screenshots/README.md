# Screenshots Directory

Place your screenshots here with the following names:

1. `baseline-workflow.png` - ZAP Baseline Scan workflow execution
2. `full-scan-workflow.png` - ZAP Full Scan workflow execution
3. `github-settings.png` - GitHub Actions workflow permissions configuration
4. `scan-results.png` - ZAP scan summary results
5. `security-issues.png` - GitHub issues created by ZAP scan
6. `artifacts.png` - Generated artifacts in GitHub Actions
7. `workflow-permissions.png` - GitHub Actions workflow permissions settings
8. `workflow-steps.png` - Complete workflow execution steps
9. `report-sample.png` - Sample ZAP scan report

## How to Capture Screenshots

### 1. Baseline Workflow
- Go to: https://github.com/NamgyelHuk708/SWE302_PA4c/actions
- Click on a successful "OWASP ZAP Baseline Scan" run
- Take screenshot of the workflow execution

### 2. Full Scan Workflow
- Go to: https://github.com/NamgyelHuk708/SWE302_PA4c/actions
- Click on "OWASP ZAP Full Scan" (or manually trigger it)
- Take screenshot of the workflow execution

### 3. GitHub Settings
- Go to: https://github.com/NamgyelHuk708/SWE302_PA4c/settings/actions
- Take screenshot of the Actions settings page

### 4. Scan Results
- Click on any workflow run
- Scroll to the ZAP scan step
- Take screenshot of the summary results

### 5. Security Issues
- Go to: https://github.com/NamgyelHuk708/SWE302_PA4c/issues
- Take screenshot showing issues #5 and #6

### 6. Artifacts
- In any workflow run, scroll to bottom
- Take screenshot of the "Artifacts" section

### 7. Workflow Permissions
- Go to: Settings → Actions → General
- Scroll to "Workflow permissions"
- Take screenshot

### 8. Workflow Steps
- In a workflow run, show all the steps expanded
- Take screenshot

### 9. Report Sample
- Download one of the HTML artifacts
- Open it and take screenshot of the report
